Your lab equipment is located remotely and you will access it in two distinct ways:
  • For the first method, you will access the remote routers and switches using their console ports. To do this, you will connect to a remote console server (also known as a terminal server) that provides serial connections to the console ports of the lab routers and switches. You can accomplish this through either an SSH session or a web link to the console server.
  • For the second method, you will connect to the remote routers and switches using a VPN tunnel. This provides access via a VPN router to the same network to which your workgroup switch connects. This second method sends packets via an encrypted tunnel across the Internet.
