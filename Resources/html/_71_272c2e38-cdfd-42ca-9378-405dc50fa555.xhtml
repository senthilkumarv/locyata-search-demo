Task 1: Update Your Workgroup Configurations
In this task, you will download a new supplemental configuration to your workgroup switch from the TFTP server. This supplemental configuration may introduce a problem that will cause loss of network connectivity, so you will troubleshoot to isolate and correct the problem.
Activity Procedure
Complete these steps:
Step1
Shut down your workgroup switch port Fa0/12, which connects to core switch B.
Step2
Ensure connectivity with the TFTP server. Ping the TFTP server (10.1.1.1) from your workgroup PC/router.
Step3
Ensure connectivity with the TFTP server. Ping the TFTP server (10.1.1.1) from your workgroup switch.
If either of the pings is unsuccessful, contact your instructor.
Step4
From your workgroup switch, download the supplemental configuration from the TFTP server into the running configuration of your workgroup switch. The name of the file to download is x-wg_sw-config-lab2-4.txt.
Step5
Type exit from the privilege EXEC prompt and ensure that your switch banner reads:
************** wg_sw-config-lab2-4 ***********************
Step6
Now can you ping the TFTP server from your workgroup PC/router?
Your ping should fail, and you should no longer have connectivity with the TFTP server.
Step7
Without utilizing the show run command, use the troubleshooting guidelines and commands that are discussed in the corresponding module to gather symptoms, isolate the problem, and correct the problem. Use the table in the Job Aids section of this lab activity to document the troubleshooting process.
Step8
Once you have isolated and corrected the problem, test connectivity by successfully pinging the TFTP server from the PC/router.
Step9
Save your running configuration to NVRAM.
Step10
Notify your instructor that you have completed the activity.
Activity Verification
You have completed this task when you attain this result:
  • Both your workgroup switch and workgroup PC/router are able to ping the TFTP server.
