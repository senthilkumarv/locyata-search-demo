Task 2: Test Connectivity to Your Assigned Remote Network
You will unsuccessfully test with the ping command the connectivity to your given remote network, which can only be reached through the serial interface that you just configured. You will then use various Cisco IOS commands to investigate the reason why you cannot reach the network.
Activity Procedure
Complete these steps:
Step1
Enter the ping remote_host command by using the assigned IP address of the remote host from the Serial WAN Information table in this lab activity. Your output should be similar to the following example:
RouterX#ping 192.168.21.200

Type escape sequence to abort.
Sending 5, 100-byte ICMP Echos to 192.168.21.200, timeout is 2 seconds:
U.U.U
Success rate is 0 percent (0/5)
Step2
Enter the traceroute remote_host command, using the same IP address that you used in Step 1. Your output should be similar to the following example:
RouterX#traceroute 192.168.21.200

Type escape sequence to abort.
Tracing the route to 192.168.21.200

   1 172.20.21.254 0 msec 4 msec 0 msec
   2 172.20.21.254 !H  *  !H
Step3
The output should indicate that the packets are being sent to the “Internet” IP address via FastEthernet 0/1.
Step4
Enter the command show ip route to view the current information that is held in the route table. Your output should be similar to the following example:
RouterX#show ip route

Notice in the example the two lines that are in boldface. These lines indicate that the only place that the router can send packets with destination addresses that are not found on directly connected networks is via the default route. Recall that the default route is indicated using 0.0.0.0. This default route points toward the “Internet” out the FastEthernet0/1 interface; the result of the previous Cisco SDM lab activity.
Activity Verification
You have completed this task when you attain these results:
  • You have observed, using the traceroute command, where your packets were being sent.
  • You have observed, using the show ip route command, that there is no entry in the routing table that matches the network that you were trying to reach. Also, the routing table has an entry for forward “unknown” destinations, known as the gateway of last resort.
