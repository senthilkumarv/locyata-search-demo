Task 2: Replace the Existing Static Route and Test Connectivity
In this task, you will remove the static route that was configured in a prior lab. You will also test connectivity to a remote network learned via the RIP routing protocol.

Step2
Notice that there are more network entries learned via RIP updates. These are indicated in the display with an “R.” However, a static route is still being used as the entry for the route to 192.168.2x.0 (where x represents your pod number) network. This is indicated with an “S.” This route, therefore, does not take advantage of the dynamic updates that are available by using RIP. Recall that the routing table uses the administrative distance to determine which route should populate the route table. The value for RIP is 120 and for a static route is 1.
Step3
Enter the conf terminal command to enter the global configuration mode.
Step4
Enter the command no ip route 192.168.2x.0 255.255.255.0 10.140.10.1 to remove the static route entry from the configuration.
Step5
Enter the end command to exit the configuration mode.
Step6
Enter the show ip route 192.168.2x.0 command to display only the information for the specified route. Your output should be similar to the following example:

Step7
Enter the traceroute 192.168.22x.200 command to use the ICMP protocol to follow the path that is taken to reach the host on the network. Your output should be similar to the following example:

Step8
Enter the command to save your configuration to the startup-config file.
Activity Verification
You have completed this task when you attain these results:
  • You have removed the static route that was configured in a prior lab.
  • You have verified the removal by using the show ip route command.
  • You have validated reachability to the network by using the traceroute command.
  • You have saved your running configuration to the startup-config file.
