Task 5: Configure the Rapid PVST Protocol
In this task, you will configure the Rapid-PVST protocol, configure the second trunk port on your workgroup switch so that it trunks to core switch B, and observe the Rapid PVST convergence when a loop is created.
Activity Procedure
Complete these steps to configure the Rapid PVST protocol on your workgroup switch:
Step1
Have the instructor verify that the interface on core switch B that connects to your workgroup switch is configured properly for trunking. (The instructor may need to enter the no shutdown command on this interface.) Make sure that the Fa0/12 interface on your workgroup switch is still shut down.
Step2
Enable the Rapid PVST protocol on your workgroup switch.
Step3
Set the speed of your workgroup switch port Fa0/12, (which connects to Core Switch B), to 100 Mb/s.
Step4
Set the duplex setting of your workgroup switch port Fa0/12, (which connects to Core Switch B), to full duplex.
Step5
Set your workgroup switch port Fa0/12, (which connects to Core Switch B), to trunk mode. Step6
Enter the no shutdown command on your workgroup switch port Fa0/12 (which connects to Core Switch B).
Step7
Enter the command to determine the spanning-tree state of the VLAN that you created earlier. Which interfaces are in the forwarding state for the VLAN that you created?_______________________________________________________________________________
Your workgroup switch port Fa0/2 connects to your workgroup PC/router and Fa0/11 connects to Core Switch A. These ports should be in the forwarding state for the VLAN that you created.
Step8
Keep your workgroup switch console session active, and open a second console session to the workgroup PC/router. (You need two open sessions to the lab equipment to accomplish this step.)
Step9
From your workgroup PC/router, use Telnet to connect to the core switches and repeat Step 7 from core switch A and core switch B.
The command to use Telnet from your PC/router is telnet 10.1.1.x. The IP address for core switch A is 10.1.1.2, and the IP address for core switch B is 10.1.1.4. The vty password for the core switches is “cisco”. You do not need to enable mode privileges on the core switches.

Step11
While keeping the two console sessions active (one to your switch and one to your PC/router), from your workgroup PC/router, perform an extended ping to the core router (10.x.x.3, in which x is your assigned VLAN number) with a count of 45000. Is the ping successful? Your output should look like the following:
RouterA# ping
Protocol [ip]:
Target IP address: 10.1.1.3
Repeat count [5]: 45000
Datagram size [100]:
Timeout in seconds [2]:
Extended commands [n]:
Sending 45000, 100-byte ICMP Echos to 10.1.1.3, timeout is 2 seconds:
You should see continuous successful ping replies from the core router. The current path from your switch to the core router should be via the port that connects your workgroup switch to core switch B (fa0/12). If not, do not proceed to the next step; instead, troubleshoot the problem or ask your instructor for help.
Step12
At your workgroup switch, shut down Fa0/11, which is the interface that connects your workgroup switch to the core switch A. Step14
Stop the extended ping from your workgroup PC/router to the core router by pressing Ctrl-Shift-6, then Ctrl-Shift-6 again.
Step15
Save your configuration to NVRAM, using copy run start.
Step16
Notify your instructor that you have completed the activity.
Activity Verification
You have completed this activity when you attain these results:
  • You have configured a second trunk port on your workgroup switch to trunk to core switch B.
  • You have observed an extended ping to the core router and shut down the forwarding trunking port to observe a break in the pings.
