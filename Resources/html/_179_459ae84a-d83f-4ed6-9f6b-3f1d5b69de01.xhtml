Task 1: Modify the Configuration Register
In this task, you will change the value of the configuration register and observe how this is displayed. You will then restore the configuration register to the value that it had at the start of this task.

Step6
You will see that your new value will not be active until the next reload.
Step7
You can (optionally) enter the show running-config command to look for the config-register parameter; however, it will not be displayed, because it is not part of the running configuration.
Step8
Enter the necessary commands to restore your configuration register to the value that you recorded in Step 3. When you have done this, you should enter the show version command and verify that the configuration register has been restored to its original value.
Step9
It can sometimes seem confusing when viewing output to distinguish which display is the running configuration and which is the startup configuration.

Step11
Notice that the output starts with the words “Building configuration.” This is because the running configuration is not a file; it is the stored parameter values that are within the executing Cisco IOS program.
Step12
Enter the show startup-config command, and use q to quit the output after the first screen is displayed. Your output should be similar to the following example:
RouterX#sh startup-config
Using 2170 out of 245752 bytes

Notice that the output of the show startup-config command displays the words “Using 2170 out of 245752 bytes,” which indicates that a certain amount of the NVRAM is being used to hold the configuration file.
Activity Verification
You have completed this task when you attain these results:
  • You have observed and recorded the current value of the configuration register.
  • You have modified the configuration register value, displayed the output of the show version command, and identified that it had been changed but that this change would not be active until after the router was restarted.
  • You have returned the configuration register to its original value.
  • You have displayed and identified the differences in the output between showing the running configuration and the startup configuration when using the show commands.
