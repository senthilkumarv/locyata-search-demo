Task 3: Using the Cisco IOS CLI Commands to Control Telnet and SSH Sessions
In this task, you will practice the initiation, suspension, and resumption of Telnet and SSH sessions from the Cisco IOS CLI. Use the username and password netadmin during this activity. You will also increase your workgroup switch vty line idle timeout to 30 minutes for the duration of this activity.
Activity Procedure
Complete these steps:
Step1
From the console of your workgroup router, open a Telnet session to your assigned workgroup switch, 10.x.x.11, using the telnet ip_address command.
Step2
Enter the command to get to the enable EXEC prompt. Your output should be similar to the following example:
RouterX#telnet 10.10.10.11
Trying 10.10.10.11 ... Open 
 
**********  Warning   *************
Access to this device is restricted to authorized persons only!
Un-authorized access is prohibited. Violators will be prosecuted. 
 
**************************************************************
User Access Verification 
 
Username: netadmin
Password:
SwitchX>enable
Password:
SwitchX#
Step3
Enter the command conf t to get to the global configuration prompt.
Step4
Enter the command line vty 0 15 to get to the vty line configuration mode.
Step5
Enter the command exec-timeout 30 to extend the idle timer period on your switch vty lines to 30 minutes.
Step6
Return to the EXEC prompt by entering the command end. Your output should be similar to the following example:
SwitchX#conf t
Enter configuration commands, one per line.  End with CNTL/Z.
SwitchX(config)#line vty 0 15
SwitchX(config-line)#exec-timeout 30
SwitchX(config-line)#end
SwitchX#
Step7
Enter the escape sequence (simultaneously hold down the <CTRL><SHIFT>6 keys then release and press x), to suspend the session and get to the RouterX# prompt.
Step8
Enter the command show sessions to display the currently active sessions. Your output should be similar to the following example, with the exception that the escape sequence has been indicated in boldface.
SwitchX#<cntrl+shift+6,x>
RouterX#show sessions
Conn Host                  Address               Byte  Idle Conn Name
*  1 10.10.10.11           10.10.10.11              0     0 10.10.10.11
RouterX#
Step9
Enter the command ssh –l netadmin ip_address to open a second connection to your workgroup switch from your router using the SSH protocol. You will need to enter the password that is associated with the username “netadmin.” Your output should be similar to the following example:
RouterX# ssh –l netadmin 10.10.10.11 
 
**********  Warning   *************
Access to this device is restricted to authorized persons only!
Un-authorized access is prohibited. Violators will be prosecuted. 
 
**************************************************************
Password: 
 
SwitchX>
Step10
Enter the escape sequence, (simultaneously hold down the <CTRL><SHIFT>6 keys then release and press x) to suspend the session and get to the RouterX# prompt.
Step11
Enter the command show sessions to display the currently active sessions. Your output should be similar to the following example, with the exception that the escape sequence has been indicated in boldface:
SwitchX><ctrl+shift+6,x>
RouterX#show sessions
Conn Host                  Address               Byte  Idle Conn Name
 1 10.10.10.11           10.10.10.11              0     4 10.10.10.11
*  2 10.10.10.11           10.10.10.11              0     0
RouterX#
Your workgroup router show session command should display two open sessions to your switch, one via Telnet and one via SSH.
Step12
Enter the command resume 1 to resume your first connection to the workgroup switch. Notice that this session returns to the enable prompt.
Step13
From your workgroup switch prompt, enter the command show users to verify the two open sessions from your router. Your output should be similar to the following example:
RouterX#resume 1
[Resuming connection 1 to 10.10.10.11 ... ]
<ENTER>
SwitchX#show users
  Line       User       Host(s)              Idle       Location
*  1 vty 0     netadmin   idle                 00:00:00 10.10.10.3
 2 vty 1     netadmin   idle                 00:00:22 10.10.10.3
Interface      User        Mode                     Idle     Peer Address 
 
SwitchX#
Your workgroup switch show users command should display two open sessions from your router, one via Telnet and one via SSH.
Step14
From your switch prompt, Telnet back to your workgroup router without prefixing the address with the word “Telnet”. Notice that you are automatically at the enable prompt on the router. Your output should be similar to the following example:
SwitchX#10.10.10.3
Trying 10.10.10.3 ... Open
**********  Warning   *************
Access to this device is restricted to authorized persons only!
Un-authorized access is prohibited. Violators will be prosecuted.
**************************************************************^C
User Access Verification
Username: netadmin
Password:
RouterX#
At this point in the activity, you have established a Telnet connection from the router to the switch and a Telnet connection back from the switch to the router. You also have an SSH connection from the router to the switch.
Step15
From your router prompt, enter the command show sessions to display any sessions that are associated with this connection. Your output should be similar to the following example:
RouterX#show sessions
% No connections open
RouterX#
The show session command output displays no open sessions because your current view is at the router via your initial Telnet connection from the router to the switch and then the telnet session back to the router. If at this point you use a single escape sequence, you will return to the Router# prompt (session 1). However, if you use two escape sequences followed by pressing x, you will return to the switch.
Step16
Enter the sequence Ctrl-Shift-6, Ctrl-Shift-6, x; notice that the x is used only once at the end. You are returned to your switch. Your output should be similar to the following example:
RouterX#<ctrl-shift-6, ctrl-shift-6, x>
SwitchX#sh sessions
Conn Host                  Address               Byte  Idle Conn Name
*  1 10.10.10.3            10.10.10.3               0     0 10.10.10.3
SwitchX#
Step17
From the switch prompt, enter the escape sequence, (simultaneously hold down the <CTRL><SHIFT>6 keys then release and press x). This escape sequence suspends the original telnet session that was initiated from the router to the switch and returns you to the RouterX# prompt.
Step18
From the router prompt, enter the show session command to view the currently open sessions. Your output should be similar to the following example:
SwitchX#<ctrl-shift-6, x>
RouterX#sh sessions
Conn Host                  Address               Byte  Idle Conn Name
*  1 10.10.10.11           10.10.10.11              0     0 10.10.10.11
   2 10.10.10.11           10.10.10.11              0     7
Observe the output. The asterisk (*) is by the number 1. This indicates that this is the last active session. Pressing the Enter key without adding any other text would resume the last active session.
Step19
Press the Enter key twice. The first resumes the connection to the switch, and the second is interpreted at the switch to resume its session to the router. You will need to press Enter a third time to get the router prompt. Your output should be similar to the following example:
RouterX#<ENTER>
[Resuming connection 1 to 10.10.10.11 ... ]
<ENTER>
[Resuming connection 1 to 10.10.10.3 ... ]
<ENTER>
RouterX#
Step20
From your router prompt, enter the sequence Ctrl-Shift-6, Ctrl-Shift-6, x to return to your switch. Your output should be similar to the following example:
RouterX#<ctrl-shift-6, ctrl-shift-6, x>
SwitchX#
Step21
From the switch prompt, close the connection to the router by using the disconnect command. Entering the command without any numerical value is interpreted as closing the last created connection. You will need to confirm your requested action. Your output should be similar to the following example:
SwitchX#disconnect
Closing connection to 10.10.10.3 [confirm]
SwitchX#
Step22
From your switch prompt, use the escape sequence Ctrl-Shift-6, x to return to your router, and at the router prompt, enter the show sessions command. Your output should be similar to the following example:
SwitchX#<ctrl-shift-6, x>
RouterX#show sessions
Conn Host                  Address               Byte  Idle Conn Name
*  1 10.10.10.11           10.10.10.11              0     1 10.10.10.11
   2 10.10.10.11           10.10.10.11              0    39
The show session command output displays the two original sessions from your router to your switch are still open.
Step23
Use the disconnect command to close both connections to the switch. Your output should be similar to the following example:
RouterX#disconnect 1
Closing connection to 10.10.10.11 [confirm]
RouterX#disconnect 2
Closing connection to 10.10.10.11 [confirm]
RouterX#
Step24
Close your SSH connection to your workgroup router by using the logout command, and disconnect your VPN connection to your workgroup.
Activity Verification
You have completed this task when you attain these results:
  • You initiated Telnet connections between your workgroup router and switch.
  • You initiated SSH connection between your workgroup router and switch.
  • You used the show sessions command to identify current connections and their values, including active session and session numbers.
  • You used the show users command to identify currently connected users to your workgroup router and switch.
  • You used the escape sequence to suspend the connection (session) that you were using (active).
  • You used the resume command to choose which of your open connections (sessions) you would use.
  • You used disconnect and logout to close all connections.
  • You terminated the VPN tunnel from your PC to your remote workgroup.
