Task 1: Add Password Protection to Console Port and Vty Lines
After the initial configuration of the switch in which you defined passwords for the vty lines, two potential security holes exist. First, a security breach is possible when the vty lines have the login process that is deactivated and the password is too simple. Second, security can be breached when the console port is not protected by any password.
Activity Procedure
Complete these steps:
Step1
Connect to your remote workgroup switch via the console server (or other available method), and enter the necessary commands and passwords to get to the enable EXEC prompt.
Step2
At the user EXEC prompt, enter the command enable followed by the enable password for your switch.
Step3
At the privileged EXEC prompt (sometimes called the “enable prompt”) of your assigned switch, enter config t.
Step4
Access the console port configuration by entering the command line console 0.
Step5
In the line console configuration mode, use the password “sanjose” for the console line. Enter the command password sanjose.
Step6
Enter the command login, which will require a password to be supplied to access the switch via the console in the future.
Step7
Enter the command line vty 0 15.
Step8
Enter the command login, which will be applied to all 16 lines (0 through 15).
Step9
Enter the command password sanjose, which will be applied to all 16 lines (0 through 15).
Step10
Enter the command end, which will return you to the enable EXEC prompt.
Step11
Enter the show running-config command and observe the output to see that you have correctly configured line console 0 and vty lines 0 through 15. Your output should be similar to the following example, where the line configuration is shown in boldface. You will observe that the passwords for both the line console and vty lines are stored in cleartext.
SwitchX# show running-config
..
..Text omitted
..
!
line con 0
   password sanjose
   login
line vty 0 4
   password sanjose
   login
line vty 5 15
   password sanjose
   login
!
end
Step12
You will now test your configured password by logging out of and back into the switch via the console.
Step13
Enter the command logout.
Step14
Press the Enter key to get a password prompt.
Step15
Supply the password that you just configured to get to the user EXEC prompt.
Step16
Enter the command and password to get to the enable EXEC prompt.
Step17
Your output for Steps 12 though 15 should be similar to the following example:
SwitchX#logout

..
..empty lines omitted
..

SwitchX con0 is now available


Press RETURN to get started.

..
..empty lines omitted
..

User Access Verification

Password:
SwitchX>enable
Password:
SwitchX#
Activity Verification
You have completed this task when you attain these results:
  • You have configured the console and vty lines to require a password.
  • You have inspected the configuration and observed that the line passwords are stored in cleartext.
  • You have successfully tested the login process and password access to the console line.
