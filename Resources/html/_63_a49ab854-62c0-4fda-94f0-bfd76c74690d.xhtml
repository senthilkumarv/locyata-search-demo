Task 4: Configure the PC/Router as a Host
In this task, you will configure your workgroup router (representing a PC) with an IP address that corresponds to its VLAN. Your PC/router can only reach other devices in a different workgroup (in a different VLAN) via the core router. Your instructor has configured the core router to support inter-VLAN routing.
Activity Procedure
Complete these steps:
Step1
Establish a connection to the lab equipment to access your workgroup PC/router console.
Step2
Choose your workgroup from the Main menu.
Step3
Choose your workgroup router from the Pod menu. If you are prompted for a console password, try a password of cisco (or consult the instructor for a password).
Step4
Enter privileged EXEC mode. If you are prompted for a privileged EXEC password, try a password of sanfran. If a password of sanfran does not work, please consult with your instructor.
Step5
Erase the startup configuration of your workgroup PC/router.
Step6
Reload your workgroup PC/router. If you are prompted to save modifications, answer N. When you are prompted to confirm reload, answer Y.
Step7
After your workgroup PC/router reboots, you will be asked if you want to enter the Configuration Dialog. Answer N. If you are asked if you want to terminate AutoInstall, answer Y.
Step8
Configure your workgroup PC/router with a hostname. Use the name that is listed in the table in the Job Aids for this lab. For example, for workgroup A, the hostname is PC_A.
Step9
Disable routing to have the router act as a PC with the no ip routing command. Do not abbreviate the command.
Step10
Assign the IP address to the first Ethernet interface (Fa0/0) of your workgroup PC/router 10.x.x.12 (in which x is your assigned VLAN number), and assign a subnet mask of 255.255.255.0. The IP address is listed in the table in the Job Aids for this lab.
Step11
Enable the Fa0/0 interface of your workgroup PC/router.
Step12
Provide a description for the interface describing the connected destination—for example, “Connection to Workgroup Switch Access Port Fa0/2”.
Step13
From privileged mode, use the show interface fa0/0 command to verify your configuration settings.
Step14
From your workgroup PC/router, ping the core router at 10.x.x.3, in which x is your assigned VLAN number. Your ping should be successful. Why?_______________________________________________________________________________
Step15
From your workgroup PC/router, ping your workgroup switch, 10.1.1.x. Your ping should not be successful. Why?_______________________________________________________________________________
Your PC/router is plugged into a switch port (Fa0/2) in your assigned VLAN (2-9). Your workgroup switch (int vlan 1) is in VLAN 1. Inter-VLAN communication requires the PC/router to have a default gateway router.
Step16
Enable inter-VLAN communication by configuring a default gateway on your workgroup PC/router that points to the core router, using the ip default-gateway 10.x.x.3 command, in which x is your assigned VLAN number. Now ping your workgroup switch.Your ping should be successful. Why?_______________________________________________________________________________
Notice that the default gateway on your workgroup switch is set to 10.1.1.3 so that your workgroup switch can return a response to the ping from devices in other VLANs via the core router. If the default gateway is not present in your workgroup switch configuration, add it by using the ip default-gateway 10.1.1.3 command in global configuration mode.
Step17
Save your running configuration to NVRAM.
Step18
Proceed to Task 5.
Activity Verification
You have completed this task when you attain these results:
  • You have assigned your workgroup PC/router with the proper identity and IP addresses.
  • You have assigned a default gateway to your workgroup PC/router.
  • You have pinged devices in other VLANs to verify connectivity.
