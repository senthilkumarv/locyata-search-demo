Task 2: Enable Serial Connections on the Workgroup Router
In this task, you will assign an IP address to your serial interfaces. You will verify that these interfaces have the proper serial encapsulation, that your router only has connectivity with directly connected devices, and that you cannot reach the remote core router IP address of 10.1.1.3. You will establish connectivity in the next task.
Activity Procedure
Complete these steps on the workgroup router to enable the serial connections:
Step1
On the workgroup router, remove the PPP encapsulation from the interface that connects to the core router, S0/0/0. This returns the S0/0/0 interface to the default encapsulation, HDLC.
Step2
Verify that the S0/0/0 interface of the workgroup router is configured for HDLC and that its IP address matches the address that is listed in the IP Addresses table in this lab activity. For example, for workgroup A, the address is 10.140.1.2.
Which command can you use to verify both the encapsulation and IP address of the S0/0/0 interface?
_______________________________________________________________________________
Step3
Ping the core router serial interface that is directly connected to your workgroup router. Refer to the IP Addresses table in this lab activity for the correct IP address. For example, for workgroup A, the address is 10.140.1.1.
The ping should work. Why?
_______________________________________________________________________________
The ping is successful across the directly connected network between your workgroup router and the core router.
Step4
Ping the core router at 10.1.1.3.
The ping should not work. Why not?
_______________________________________________________________________________
The ping is unsuccessful because your workgroup router has not learned of any remote networks that are not directly connected.
Step5
View your IP routing table to see all the networks that are listed in the table.

Step6
To configure the serial connection between you and your lab partner’s router, verify whether a DCE or DTE cable is connected on your second serial interface (S0/0/1) by using the show controllers serial interface command. (Notice that there is a space between the word serial and the interface parameter.)
Step7
If your second serial interface (S0/0/1), which connects to your partner workgroup router, is DCE, assign a clock rate of 64000.
DTE interfaces do not require a clock rate to be set.
Step8
Configure the IP address of the second serial interface (S0/0/1) going to your partner’s router with the IP address that is listed in the IP Addresses table.
Step9
Enter the no shutdown command on your second serial interface.
Step10
Ping the second serial interface (S0/0/1) of your partner router that is directly connected to your workgroup router. Refer to the IP Addresses table for the correct IP address.
The ping should work. Why?
_______________________________________________________________________________
A successful ping requires the S0/0/1 interface of your partner router S0/0/1 to be configured correctly.
Step11
Proceed to Task 3.
Activity Verification
You have completed this task when you attain these results:
  • You have removed the PPP encapsulation that was configured previously on the router.
  • You have assigned an IP address to your serial interfaces.
  • You have verified connectivity with your directly connected serial interface neighbor routers.
  • You have verified that you cannot reach the core router IP address of 10.1.1.3 by unsuccessfully pinging the router.
